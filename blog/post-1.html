用 Python 开发命令行待办事项管理器

从零实现一个支持增删改查、JSON 持久化、日期排序的 CLI 工具

项目背景

这是我的 Python 课程作业，要求开发一个命令行待办事项管理器。项目需要实现核心 CRUD 功能、JSON 持久化存储、截止日期自动排序，并做好异常处理与代码规范。

功能实现

1. 核心功能

  • 新增待办：输入标题（必填）、描述、截止日期（YYYY-MM-DD 格式）
  • 查看待办：支持全部 / 未完成 / 已完成三种筛选，按截止日期升序排列
  • 修改待办：按 ID 选中后可修改任意字段，直接回车保留原值
  • 删除待办：按 ID 选中并二次确认后删除

2. JSON 持久化

程序启动时从 todos.json 加载数据，每次操作后实时保存，关闭程序数据不丢失。数据结构示例：

[
  {
    "id": 1,
    "title": "完成作业",
    "description": "Python 命令行项目",
    "due_date": "2026-06-30",
    "done": false
  }
]

3. 截止日期排序

使用 sorted() 函数配合 lambda，按 due_date 字段升序排列，保证紧急任务优先显示。

4. 异常处理

  • 日期格式不合法时循环提示重新输入
  • 标题为空时拒绝新增
  • 输入非数字 ID 或不存在的 ID 时给出提示
  • 随时可按 Ctrl+C 取消当前操作回到主菜单

代码结构

全部功能拆分为独立函数，结构清晰：

  • load_todos() / save_todos() — JSON 读写
  • add_todo() / list_todos() / edit_todo() / delete_todo() — 四个核心操作
  • input_date() — 日期校验循环
  • find_by_id() / require() — 统一的 ID 查找与非空校验
  • sorted_by_date() — 截止日期排序
  • main() — 菜单循环

AI 使用反思

本项目借助 AI（Claude）生成了初版代码框架。有帮助的地方：

  • 快速生成函数拆分方案，节省了搭骨架的时间
  • 异常处理覆盖更全面，提醒了边界情况

需要批判性使用的地方：

  • AI 注释有时描述"是什么"而非"为什么"，需要清理
  • AI 倾向于加入过多防御性代码，需结合实际需求取舍
  • 代码逻辑仍需自己逐行理解，不能盲目复制粘贴

结论：AI 是加速器，不是替代品。它适合生成骨架，但代码的正确性、合理性、风格一致性仍需开发者自己把关。
