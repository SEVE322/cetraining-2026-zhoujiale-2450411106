# 个人待办事项管理器

一个基于 Python 命令行的待办事项管理工具，支持增删改查、JSON 持久化与截止日期自动排序。

---

## 运行环境

- Python 3.7 及以上
- 无需安装任何第三方库（仅使用标准库）

---

## 快速开始

```bash
python todo.py
```

程序启动后进入主菜单，数据自动保存在同目录的 `todos.json` 文件中。

---

## 功能说明

| 菜单选项 | 功能 | 说明 |
|----------|------|------|
| 1 | 新增待办 | 输入标题（必填）、描述、截止日期（YYYY-MM-DD），自动标记为未完成 |
| 2 | 查看待办 | 支持三种筛选：全部 / 只看未完成 / 只看已完成，结果按截止日期升序排列 |
| 3 | 修改待办 | 按 ID 选中后可修改任意字段，直接回车保留原值，支持切换完成状态 |
| 4 | 删除待办 | 按 ID 选中并二次确认后删除，防止误操作 |
| 0 | 退出 | 退出程序（数据已实时保存，不会丢失） |

### 异常处理

- 日期格式不合法时循环提示重新输入
- 标题为空时拒绝新增
- 输入非数字 ID 时给出提示
- 删除或修改不存在的 ID 时给出提示
- 随时可按 `Ctrl+C` 取消当前操作回到主菜单

---

## 数据格式（todos.json）

```json
[
  {
    "id": 1,
    "title": "完成作业",
    "description": "Python 命令行项目",
    "due_date": "2026-06-30",
    "done": false
  }
]
```

## AI 使用反思

本项目在开发过程中借助 AI（Claude）辅助生成了初版代码框架与结构设计。

**有帮助的部分：**
- 快速生成了函数拆分方案，让我可以直接思考哪些设计合理、哪些需要调整
- 异常处理的覆盖场景比自己手写更全面，提醒了"删除不存在 ID""输入非数字 ID"等边界情况

**需要批判性使用的部分：**
- AI 生成的注释有时过于啰嗦，描述"是什么"而非"为什么"，需要手动清理
- AI 倾向于加入过多防御性代码，而实际上程序内部逻辑不需要这些，需要结合实际需求取舍
- 理解代码逻辑仍需自己逐行阅读，不能盲目复制粘贴——比如 `todos` 列表是按引用传递的，AI 最初写法多余地 return 了列表，需要自己判断是否必要

**结论：** AI 是加速器，不是替代品。它适合生成骨架，但代码的正确性、合理性、风格一致性仍需开发者自己把关。
